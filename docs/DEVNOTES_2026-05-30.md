# CoupLB Developer Notes: 2026-05-30 Fixes

This note documents four bugs fixed after a source review, plus why they did not
show up in many existing simulations.

## Fixed Issues

### Self-periodic IBM force exchange

`Streaming::exchange_force_dim()` used the correct reverse-force exchange for
MPI neighbor ranks, but the single-rank periodic special case accumulated the
low ghost layer onto the low interior face and the high ghost layer onto the
high interior face.

For a self-periodic dimension, those ghost layers represent the opposite side of
the domain. The correct mapping is:

- low ghost layer -> high interior face
- high ghost layer -> low interior face

This only affects IBM particles whose spreading stencil crosses a periodic
boundary in a dimension handled by a single MPI rank.

### LBM advance during setup

`FixCoupLB::setup()` called `post_force()`. That initialized IBM coupling, but it
also advanced the LBM state before LAMMPS executed the first timestep. As a
result, step-0 profile or VTK output could already contain one fluid update.

Setup now initializes IBM forces for the first fluid step without calling
`lbm_step()`. Step-0 output is therefore the actual initial fluid state.

### PVD time with `md_per_lb > 1`

VTK series files store LAMMPS timestep numbers in `vtk_steps`, but `.pvd` output
used `step * dt_LBM` for the time coordinate. Since `dt_LBM = update->dt *
md_per_lb`, the time axis was too large by `md_per_lb`.

The `.pvd` writer is now called with the MD timestep, so `timestep * update->dt`
matches LAMMPS time.

### Profile output under x/z decomposition

`write_profile()` wrote only the y-profile assembled by world rank 0's x/z
column. This was fine for 1-rank and y-only decompositions, but it was not a
global x/z average when MPI also decomposed x or z.

Profile output now reduces y-bin sums and cell counts over all ranks before
rank 0 writes the profile.

## Why Many Simulations Were Unaffected

The bugs were conditional rather than always-on:

- The self-periodic force bug requires an IBM marker stencil to cross a periodic
  boundary and that dimension must be periodic on a single MPI rank. Markers
  away from periodic edges do not exercise it.
- The setup bug is most visible in step-0 output or very short transients. Long
  driven flows quickly make the extra initial fluid update negligible compared
  with the full run length.
- The PVD bug only changes visualization time metadata, not the simulated fluid
  or particle state. It also only matters when `md_per_lb > 1`.
- The profile bug only affects diagnostics when the simulation is decomposed in
  x or z and the field is not homogeneous over those directions. Single-rank,
  y-only, and laterally homogeneous Poiseuille-style tests are effectively
  insensitive.

These are still correctness bugs because they can silently bias boundary IBM
coupling, early-time diagnostics, visualization time axes, or parallel profile
diagnostics in the affected regimes.
